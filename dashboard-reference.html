1. Vue d'ensemble

Le système est une machine de prospection B2B full-automatisée avec deux pipelines distincts qui partagent la même base SQLite.

Pipeline Maps (Ancien)

Google Maps → Scrape → Audit PageSpeed → Copywriter → Email → Resend/Brevo

Cible : PME locales avec email valide trouvé sur Maps

Pipeline Sniper (Nouveau)

6 sources d'annonceurs → Enrichissement CEO + SMTP → Scoring → Email généré → Envoi step 1 → Rapport step 2

Cible : Entreprises qui dépensent déjà en pub, donc convaincues du ROI digital

2. Section Sniper — Sources

ADS (Google/Bing)
  • Ce qu'elle fait : Scrape les pages de résultats Google via Playwright
  • Configuration : headless, IPv4 forcé (--disable-ipv6)
  • Pagination : Jusqu'à épuisement (2 pages vides consécutives, plafond 100 pages)
  • Volume : ~3-6 annonceurs par page
  • Qualité : Leads des pages profondes = meilleurs (petit budget, pas d'équipe technique)
  • Contrainte : VPN français actif requis
  • Banque : 125 mots-clés, 10 secteurs, rotation par seed date
Force : Cible qualifié
Limite : VPN obligatoire
FB ADS (Meta Ad Library)
  • Ce qu'elle fait : Appel API officielle /ads_archive
  • Token : Graph Explorer (~1-2h court terme) ou 60 jours (long-lived via App Secret)
  • Champs : page_name, website, fan_count, ad_creative
  • Filtrage : Leads sans site → tag creation si fan_count≥100 et pub active depuis ≥7 jours
Force : Données officielles
Limite : Token expire
E-COM
  • Ce qu'elle fait : Scrape les entreprises e-commerce françaises via codes NAF
  • Détection : Shopify / WooCommerce / Prestashop
  • Cible : Boutiques avec panier moyen élevé mais site non optimisé
Force : Panier élevé
Limite : Détection CMS
JOBS
  • Ce qu'elle fait : Scrape les offres d'emploi (développeur, webmaster, marketing digital)
  • Signal fort : Chercher quelqu'un = pas d'équipe interne = besoin de prestataire
Force : Signal intent
Limite : Volume faible
MAPS (Google Maps)
  • Source du pipeline classique, toujours actif
  • Scrape les établissements locaux sur Google Maps pour un keyword + ville
  • ~60-120 leads par recherche
  • Signal : présence Maps = clientèle locale existante
  • Email trouvé directement sur leur site
  • Leads filtrés par validation SMTP
  • Pipeline séparé du Sniper mais base de données partagée
Force : Volume local
Limite : Email générique
BODACC
  • Ce qu'elle fait : Scrape le Bulletin Officiel des Annonces Civiles et Commerciales
  • Détection : Nouvelles nominations de dirigeants
  • Signal : Nouveau CEO = nouveau regard sur les outils digitaux
Force : Signal changement
Limite : Données brutes

3. Section Sniper — Pipeline d'Enrichissement

Phase 1 — Extraction

Selon la source, collecte les domaines annonceurs bruts.

Phase 1.5 — Pré-filtre
  • Mesure TTFB (Time To First Byte)
  • Détecte GTM/GA (présence = tracké = maturité digitale)
  • Rejette les grosses marques (score de chaleur 0-12)
  • Pour source=ads : min_score=0 (annonceurs ont forcément GTM)
Phase 1.5 — Enrichissement Contact
  • contact_finder : Scrape page /contact, footer, mentions légales
  • ceo_finder : Google dork + Ollama local
  • email_permutations : Génère prenom.nom@domaine.fr, etc.
  • smtp_validator : Vérifie SMTP sans envoyer
Phase 2 — PageSpeed + Wappalyzer

Analyse perf mobile/desktop, LCP, détecte CMS.

Phase 3 — Scoring
  • score_lead() → Tag urgence + niveau 1-5
  • Tags : perf / securite / perf+securite / creation
  • Rejeté si score insuffisant
Phase 4 — DB

insert_lead() avec déduplication par nom+ville, téléphone, site_web (clé naturelle Sniper).

Phase 5 — Génération email automatique

generate_sniper_emails_batch() se déclenche automatiquement après le scoring. Copywriter adapte le ton selon le tag et le mode (email valide / catch-all / sans contact).

5. Section Maps — Pipeline Classique

C'est le pipeline originel du système, toujours actif et complémentaire au Sniper. Il cible les PME locales référencées sur Google Maps — artisans, commerces, professions libérales — qui ont une présence Maps mais un site web non optimisé.

Scraper (scraper/)
  • Playwright + patchright navigue sur Google Maps
  • Input : mot-clé + ville
  • Extrait : nom, adresse, téléphone, site web, email (depuis le site), note, nombre d'avis, catégorie, lien Maps
  • Limite naturelle : ~60-120 résultats par recherche Maps selon la densité locale
  • Déduplication : par nom+ville ou téléphone
Validation email
  • email_validator.py vérifie via SMTP sans envoyer
  • Leads filtrés selon email_valide = 'Valide'
Audit (auditeur/)
  • PageSpeed Insights API (mobile + desktop) : LCP, CLS, FID, score performance, score SEO
  • Google My Business : vérifie photos, horaires, réponses aux avis
  • Score urgence (0-10) : combine perf mobile + ancienneté site + CMS détecté
Copywriter (copywriter/) — 8 situations S1 à S8
Code Situation Angle email
S1Site rapide, bon GMBPas de lead (trop bien)
S2Site lent, pas de GMBDouble problème = urgence
S3Site lent, bon GMBPerf = argent perdu
S4Site rapide, mauvais GMBRéputation en danger
S5Pas de siteProposition création
S6Site lent + vieux CMSMigration + perf
S7Bon score mais SEO faibleVisibilité = croissance
S8Site récent mais non optimiséQuick wins possibles

Templates A/B/C/D : 4 tons différents (direct/soft/technique/ROI). Le copywriter choisit selon S1-S8 + profil de l'entreprise.

Envoi (envoi/)
  • Resend : provider principal, 100 emails/jour gratuit, tracking webhooks (open, click, bounce, delivered)
  • Brevo : fallback si Resend épuisé ou en erreur
  • Retry automatique avec backoff exponentiel
  • Fenêtre d'envoi configurable (par défaut 9h-18h)
Planification (dashboard/pipeline/)
  • Campagnes planifiées dans planned_campaigns avec date_planifiee
  • Batch scheduler : remplit les slots disponibles selon quota restant
  • Auto-approval après 5h si pas de validation manuelle (configurable)
  • Relances automatiques (sequence_worker) selon délai configuré
Suivi et séquence
  • Webhooks Resend → email_events : open, click, bounce, spam
  • IMAP polling toutes les 15 min pour détecter les réponses
  • Notification Telegram pour chaque lead qui répond
  • Step 2 déclenché manuellement ou par validation Telegram (callback sniper_step2_ID)

Limites Maps

  • Email souvent générique (contact@, info@) — moins précis que Sniper
  • Pas de ciblage CEO
  • Un seul email par lead (pas de step 2 structuré)

10. Ce qu'on peut et ne peut pas faire

Peut faire
  • Scraper tous les secteurs en France simultanément
  • Collecter jusqu'à ~100 leads qualifiés par jour (avec VPN + scraping auto)
  • Enrichir automatiquement CEO, email, téléphone
  • Générer un email personnalisé par lead automatiquement
  • Envoyer step 1 dans le quota quotidien
  • Tracker opens/clicks via webhooks
  • Basculer sur LinkedIn automatiquement pour les catch-all
  • Voir en temps réel le nombre de leads collectés pendant un run
  • Filtrer par qualité de contact et trier par urgence
Ne peut pas faire (encore)
  • Dépasser 100 emails/jour sans changer de plan Resend
  • Scraper Google sans IP française (VPN obligatoire)
  • Utiliser FB Ads plus de 2h sans renouveler le token (ou configurer long-lived)
  • Envoyer automatiquement sans validation humaine (step 1 auto désactivé par défaut)
  • Détecter les réponses en temps réel (poll IMAP toutes les 15 min)
