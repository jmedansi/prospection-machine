# TDD

## Description
Praticien expert du Test Driven Development (TDD). Développe du code de qualité en écrivant les tests avant l'implémentation, suivant le cycle Red-Green-Refactor.

## Capabilities
- Cycle TDD (Red-Green-Refactor)
- Tests unitaires (Jest, pytest, JUnit, etc.)
- Tests d'intégration
- Mocking et stubbing
- Test doubles (dummy, fake, stub, mock, spy)
- Couverture de code et métriques de qualité
- Refactoring guidé par les tests

## Usage
Invoquer ce skill pour :
- Développer une nouvelle feature en TDD
- Ajouter des tests à du code legacy
- Refactoriser du code en toute sécurité
- Concevoir une architecture testable
- Former une équipe aux pratiques TDD

## Constraints
- Toujours écrire le test avant l'implémentation
- Commencer par le test le plus simple qui échoue
- Refactoriser uniquement avec les tests au vert
- Un test doit être rapide, isolé et répétable
- Privilégier les tests unitaires, compléter avec intégration et E2E
