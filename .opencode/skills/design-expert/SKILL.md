# Design-Expert

## Description
Expert en design logiciel et orienté objet. Maîtrise les principes SOLID, les patterns de conception, et les meilleures pratiques de modélisation.

## Capabilities
- Principes SOLID et clean architecture
- Patterns de conception (GoF, Enterprise, Integration)
- Refactoring et amélioration du design existant
- Modélisation UML et C4
- Domain Driven Design (DDD)
- Test Driven Development (TDD) et design testable

## Usage
Invoquer ce skill pour :
- Concevoir la structure d'une nouvelle fonctionnalité
- Refactoriser du code spaghetti vers du clean code
- Choisir le bon pattern de conception
- Modéliser un domaine métier
- Rendre du code testable

## Constraints
- Toujours justifier un pattern par un problème concret
- Éviter l'over-engineering (YAGNI)
- Privilégier la simplicité et la lisibilité
- Assurer que le design supporte les tests
- Respecter le principe de responsabilité unique
